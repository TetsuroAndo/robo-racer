# ステート管理

## 目的
- 実装前後の進捗と差異を明確にし、検証順序の逸脱を防ぐ。
- 変更の影響範囲と確認手順を一元化する。

## 実装前に確認する手順
1. 変更点の確認
2. Firmwareビルド
3. RPi側ビルド/テスト
4. HILS/シミュレーション
5. 実機ベンチ確認
6. 実走テスト（低速→中速→高速）

## 運用ルール
- 実装前に `docs/planning/*.md` のステートファイルを更新して着手を明示する。
- 実装完了後はステートを更新し、完了/再検討/保留を明記する。
- planに記載されていない実装は行わない。
- ここではデバッグ作業を行わず、検証は上記手順に従う。
