# インターフェース凍結・実装ステータス（タスクA）

## 目的
RPi↔ESP32の既存プロトコル/安全設計を契約として凍結し、
タスクB（ルールベース/SLAM/安全）とタスクC（AI/データ/推論入口）が車体なしで並行開発できる状態を作る。

- `shared/proto` にIPC用メッセージ型（v1）を追加
- RPi側 Logger API の一本化（mc_core へ統合）
