# DRIVE 送信間隔ログ化ステータス

- **目的**: RPi 側の DRIVE コマンド送信間隔の最大値（worst gap）をログに残し、TTL の妥当性を実測で判断できるようにする。
- **参照**: `docs/planning/ftg_v1.md` に TTL=30ms 想定が残っており、現在の TTL=100ms との乖離があるため、実測で送信周期の信頼性を確認する。

## 状態
- **進行中**: `Sender::send` に送信間隔測定と worst gap の記録を追加し、ログ出力する実装。
- **未対応**: ログを取りつつ worst gap を外部に出力する検証や自動化なし。
- **テスト**: 現在なし（手動でログを確認予定）。
